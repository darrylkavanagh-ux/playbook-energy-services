# WHAT DEREK DUNPHY CAN COPY - BY ACCESS LEVEL

## SCENARIO 1: You Give Full GitHub Repo Access

### What Derek Can Copy:
✅ **100% of everything**
- All 19,560 lines of backend code
- All 20 forensic engine algorithms
- All database schemas
- All API endpoints
- All strategic documents
- All business models
- All pricing information

### Can He Rebuild It?
🔴 **YES - With AI help in 2-4 weeks**

Example:
1. Derek clones your repo
2. Derek says to AI: "Analyze this codebase and explain how it works"
3. AI reads all code, understands architecture
4. Derek says: "Build me a similar system"
5. AI rebuilds in 2-4 weeks with different code

### Can You Stop Him?
❌ **NO** - Once code is seen, it's known
- Copyright protects specific code (he can't copy-paste)
- Copyright does NOT protect ideas/algorithms
- He can rebuild with different code (legal)
- Patents are too slow/expensive (18+ months, £50-100k)

### Your Only Protection:
- NDA (he can't share what he saw)
- Trade secret law (if he steals to compete)
- First-mover advantage (you're already ahead)

## SCENARIO 2: Frontend-Only Public Website

### What Derek Can See:
✅ **Everything visible in browser**
- All HTML/CSS/JavaScript (public by nature)
- All page layouts and designs
- All text content and marketing copy
- All images and branding
- All user interface flows

### What He CANNOT See:
❌ Backend code (not loaded in browser)
❌ Database structure
❌ API implementation details
❌ Forensic algorithms
❌ Strategic documents

### Can He Copy Website Design?
🟡 **YES - But less valuable**
- Design can be copied (not protected)
- Branding/logo protected by trademark
- Content protected by copyright
- He can make "similar" site legally

### Can He Rebuild Backend From Frontend?
🟠 **PARTIALLY**
- He can see what features exist (API calls visible)
- He can't see how they work (implementation hidden)
- With AI, he could guess/rebuild (takes longer)

Example of what's visible in browser:
```javascript
// Frontend code - visible to Derek
fetch('https://api.playbook.ie/api/forensic/analyze', {
  method: 'POST',
  body: JSON.stringify({ data: evidence })
})
```

From this, Derek knows:
- ✅ You have an API endpoint `/api/forensic/analyze`
- ✅ It accepts POST requests
- ✅ It processes evidence data
- ❌ He can't see implementation (what analysis it does)
